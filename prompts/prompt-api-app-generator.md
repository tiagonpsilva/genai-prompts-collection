# Nome
API App Generator

# Descrição
Geração de uma aplicação (API)  baseado nos artefatos informados

# Instruções
```
Gerar uma aplicação back-end  para implementação de uma API RESTful baseada na estrutura dos arquivos anexados

Considere as seguintes características:
- Linguagem Python
- Framework FastAPI
- Estrutura de projeto baseada no FastAPI Best Practices
- ORM SQLAlchemy
- Arquivo requirements.txt
- Pasta com c4model (level 1 e 2) com a arquitetura da aplicação, utilizando plantUML
- Banco de dados SQLite, com dados de exemplo
- Arquivos .sql para criação das tabelas
- Arquivo postman para testes da API
- Estrutura para gravação de logs trace, debug, info, warning, error, critical
- Estrutura para geração de documentação automática (swagger)
- Diretorio para upload de arquivos
- Arquitetura RESTful, Clean Architecture
- Clean Code, Orientação a objetos, código bem comentado em português
- Testes unitários 
- Design Patterns, SOLID
- Arquivo .gitignore
- Arquivo Dockerfile, docker-compose.yml
- Estrutura para geração de iac (terraform)
- Arquivos .env, .env.example
- Possibilidade de exeção via console (com instruções)
- Possibilidade de execução via docker-compose (com instruções)
- readme.md com a descrição completa do projeto e as formas de executá-lo
- comando linux para geração de todas as pastas e arquivos necessários
```
